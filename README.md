# 🚀 Wallet Expo Hackathon Submission Repository

Welcome to the official repository for submitting your projects for the Wallet Expo Hackathon.

This repository is structured department-wise using folders. Please follow all instructions carefully to ensure your submission is accepted.

---

## 📂 Repository Structure

The repository contains the following folders:

CS/  
IT/  
CSB/  
CSAI/  

Each folder represents a specific department.

👉 You must submit your project ONLY inside your respective folder.

Example:

CS/LCS20250XX/

---

## 📌 Important Rules

- Do NOT push your project to the wrong folder
- Do NOT modify or delete other participants' work
- Do NOT overwrite existing submissions
- Always create your own folder inside your department
- Follow proper Git workflow (fork → branch → PR)

---

## 🧾 Folder Naming Convention

Inside your folder, create a directory named:

roll_number

---

## 📝 Project README (MANDATORY)

Inside your project folder, include a README.md with:

- Project Overview  
- Features  
- How to Run the Project  
- Demo Link (Mandatory - submissions without demo will NOT be considered)

---

## 🔄 Submission Steps

### 1. Fork the Repository

Click the Fork button on GitHub.

---

### 2. Clone Your Fork

git clone https://github.com/your-username/Wallet-Expo.git  
cd Wallet-Expo  

---

### 3. Create a New Branch

git checkout -b roll_number  

---

### 4. Add Your Project

- Go to your department folder (CS / IT / CSB / CSAI)
- Create your roll_number folder
- Add all your project files inside it

---

### 5. Commit Changes

git add .  
git commit -m "Added project: roll_number"  

---

### 6. Push to GitHub

git push origin roll_number

---

### 7. Create Pull Request

- Go to your forked repo on GitHub
- Click "Compare & Pull Request"
- Submit your PR to the main repository

---

## ✅ Submission Checklist

Before submitting, ensure:

- Your project is inside the correct folder  
- Folder name follows the convention  
- README.md is included  
- Demo link is provided  
- Project runs without errors  
- No unnecessary files are included  

---

## ⚠️ Important Notes

- Incorrect submissions will be rejected  
- Submissions without demo links will NOT be accepted  
- Late submissions may not be considered  
- Keep your code clean and well-structured  

---
